【教程】用Hexo和Github搭建自己的博客

发表于2022-09-18|更新于2022-09-18
|阅读量:

前言

本博客是我搭建的第一个博客，采用的是Hexo + Github Pages的方式。Hexo是一款静态博客框架，选择Hexo是因为其生态比较成熟，而且没有后端对于我这种web小白来说也算是一个优点，宜上手；选择Github Pages是因为你可以免费把你的网页托管上去，省去了搭建服务器的经历和费用，可以真正实现零成本建站。
在这里，我把搭建博客的方法分享出来，希望能帮助到更多想拥有自己博客的人。
注：如果搭建中遇到问题，请善用科学上网及搜索引擎

准备工作

Github

Github

请确保你拥有一个GitHub账号，如果没有，可以现在去注册一个
GitHub是一个面向开源及私有软件项目的托管平台，因为只支持Git作为唯一的版本库格式进行托管，故名GitHub。

Git

Git 是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。
我们可以通过Git把自己的代码上传到GitHub的仓库里
下载地址： Git
Git 的安装请参考：Git 安装教程
具体的 git 学习可以访问：Pro Git 中文版

Github Pages

Github Pages 是 Github 为用户提供的服务，我们可以把静态网页托管在上面。使用 Github Pages 相当于白嫖一个服务器，免去了自己搭建服务器和写数据库的麻烦。此外还可以绑定自己的域名。
官网： Github Pages

Node.js

Hexo的运行需要依赖Node.js
下载地址： Node.js

二、用编辑器编写文章

md 全称 Markdown， Markdown 是 2004 年由 John Gruberis 设计和开发的纯文本格式的语法，非常的简单实用，常用的标记符号屈指可数，几分钟即可学会， .md 文件可以使用支持 Markdown 语法的编辑器编辑，然后将写好的文章（.md 文件）保存到 \Blog\source_posts 文件夹下即可当我们用编辑器写好文章后，可以使用 hexo g -d 命令将其推送到服务器上

现在访问你的博客就可以看见写好的文章啦！
